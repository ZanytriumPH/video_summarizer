# 5.3 升级方向项目拓展计划（Map-Reduce 多智能体协同）

## 1. 计划目标与评审口径

### 1.1 目标
将当前“单图单轮并行分析 + 融合回路”升级为“长视频可扩展的 Map-Reduce 多智能体协同架构”，核心解决：

1. 长视频输入下的上下文拥塞与质量下降。
2. 视觉 token 浪费（重复帧、静态画面重复分析）。
3. 单节点过重导致的耗时与失败率上升。

### 1.2 成功标准（评审通过条件）
1. 支持按时间窗口分片并发处理（Map）。
2. 支持分片结果汇聚生成全局总结（Reduce）。
3. 在同等输入规模下，整体时延和 token 成本显著优于现状。
4. 在测试集中，信息覆盖率与事实一致性不低于现有版本。
5. 与现有 thread_id/checkpoint/time-travel 保持兼容。

### 1.3 非目标（本期不做）
1. 不引入新的外部向量数据库（5.5 再做）。
2. 不做 HITL 中断审批（5.4 再做）。
3. 不替换前端框架（继续 Streamlit）。

---

## 2. 当前基线与升级差异

### 2.1 当前基线（已实现）
1. 工作流图：文本分析与视觉分析并行，之后融合、幻觉审查、有用性审查。
2. 已有会话与时间旅行能力：thread_id + checkpointer + timestamp 追问。
3. 单次总结入口稳定可用。

### 2.2 升级核心差异
1. 从“全量 transcript + 全量关键帧一次处理”升级为“分片并发处理”。
2. 新增角色化 Worker：Audio Agent、Vision Agent、Chunk Synthesizer。
3. 新增 Reduce 汇总器：跨分片去重、时间线重排、全局洞察归并。
4. 新增分片级质量闸门与全局质量闸门（双层控制）。

---

## 3. 目标架构设计（落地版）

## 3.1 架构分层
1. 切分层（Chunk Planner）：按时长与内容密度生成窗口任务。
2. Map 层（Worker Graph）：每个分片并发执行 Audio/Vision 分析与局部融合。
3. Reduce 层（Global Synthesizer）：汇聚所有分片输出，生成全局草稿。
4. Guardrail 层（Global Graders）：全局幻觉与有用性复审，必要时触发定点重写。
5. Persistence 层：按 thread_id 保存分片中间态和全局态。

## 3.2 拓扑（建议）
START
-> chunk_planner_node
-> map_dispatch_node（通过 Send API 分发 chunk）
-> chunk_audio_node + chunk_vision_node（并行）
-> chunk_synthesizer_node
-> collect_chunk_results_node（Fan-in）
-> global_fusion_node（Reduce）
-> hallucination_grader_node
-> usefulness_grader_node
-> END

注：若 grader 失败，优先回流 global_fusion_node；必要时可回流指定 chunk 重跑（第二期增强）。

## 3.3 数据契约（新增状态字段建议）
在现有 VideoSummaryState 基础上新增：

1. video_duration_seconds: int
2. chunk_plan: list[dict]
3. chunk_results: list[dict]
4. global_entities: list[str]
5. global_timeline: list[dict]
6. reduce_debug_info: dict
7. chunk_retry_count: dict[str, int]

chunk_plan 单项建议结构：
- chunk_id
- start_sec
- end_sec
- keyframe_indexes
- transcript_segment_indexes
- priority

chunk_results 单项建议结构：
- chunk_id
- audio_insights
- vision_insights
- chunk_summary
- evidence_refs
- token_usage
- latency_ms

## 4. 分阶段实施计划（建议 4 个迭代）

## 4.1 迭代 A：分片与分发骨架（预计 4-6 天）

### 开发项
1. 新增切分模块：
- core/workflow/video_summary/planner/chunk_planner.py
2. 新增 Map 分发节点：
- core/workflow/video_summary/nodes/map_dispatcher.py
3. 图编排改造：
- core/workflow/video_summary/graph.py

### 验收项
1. 输入视频可生成稳定 chunk_plan。
2. chunk 覆盖率 100%，无重叠越界。
3. 分发逻辑可在测试中稳定触发。

### 风险点
1. Whisper 时间戳缺失导致切分映射不稳定。
2. 关键帧时间与 transcript 时间轴偏差。

---

## 4.2 迭代 B：Chunk Worker 群落（预计 5-7 天）

### 开发项
1. 新增分片音频分析节点：
- core/workflow/video_summary/nodes/chunk_audio_analyzer.py
2. 新增分片视觉分析节点：
- core/workflow/video_summary/nodes/chunk_vision_analyzer.py
3. 新增分片融合节点：
- core/workflow/video_summary/nodes/chunk_synthesizer.py
4. 新增工具防抖策略：
- 同一 chunk 内搜索调用限流
- 同类 query 结果缓存

### 验收项
1. 每个 chunk 可独立产出 chunk_summary。
2. 分片级异常可降级，不阻塞全局流程。
3. 分片平均耗时在基线可接受范围内。

### 风险点
1. 并发度过高触发 API 限流。
2. 分片粒度过小导致上下文碎片化。

---

## 4.3 迭代 C：Reduce 全局汇总（预计 4-6 天）

### 开发项
1. 新增汇聚节点：
- core/workflow/video_summary/nodes/collect_chunk_results.py
2. 新增全局融合节点：
- core/workflow/video_summary/nodes/global_fusion_drafter.py
3. 复用并对接现有双评分器：
- hallucination_grader_node
- usefulness_grader_node

### 验收项
1. 全局报告包含“分片主线 + 跨片关系”。
2. 重复内容去重有效，时间线连续。
3. 在长视频样例上无明显信息断层。

### 风险点
1. Reduce 提示词设计不当导致“拼接感强”。
2. 分片风格漂移导致全局语气不一致。

---

## 4.4 迭代 D：持久化、回溯与前端联动（预计 3-5 天）

### 开发项
1. 在 checkpoint 中保存 chunk 级中间态。
2. time_travel 检索增加 chunk 维度定位。
3. 前端增加“分片执行可视化”与“失败分片提示”。

涉及文件：
- core/workflow/api.py
- core/workflow/time_travel.py
- services/workflow_service.py
- app.py

### 验收项
1. 可追溯任意 chunk 的分析证据。
2. 时间旅行问答可引用 chunk_summary 与原始证据。
3. 前端可展示每个 chunk 的执行状态。

## 6. 测试与验收方案

## 6.1 单元测试
1. chunk 切分正确性：边界、重叠、时长异常。
2. dispatcher 输出正确性：Send 目标与负载。
3. chunk worker 稳定性：空输入、异常输入、超时降级。
4. reduce 去重逻辑：重复实体、重复时间段、冲突结论。

## 6.2 集成测试
1. 中等时长视频（10-20 分钟）端到端链路。
2. 长视频（60 分钟以上）压力路径。
3. API 失败注入（429/500/timeout）下的降级行为。
4. thread_id 恢复后再次追问的一致性。

## 6.3 评测指标
1. 质量指标：
- Coverage@K（关键信息覆盖率）
- Hallucination Rate（幻觉率）
- User Intent Match（需求命中率）
2. 性能指标：
- P95 总时延
- 每分钟视频 token 消耗
- 平均每 chunk 耗时
3. 稳定性指标：
- 失败重试后成功率
- 超时降级命中率

## 6.4 验收门槛（建议）
1. 与现有版本相比，P95 时延下降至少 20%。
2. 每分钟视频 token 成本下降至少 15%。
3. 幻觉率不高于现有版本。
4. 时间旅行追问准确性不回退。

---

## 7. 风险与缓解策略

1. 并发引发限流：
- 缓解：全局并发阈值 + 指数退避 + 分片排队。
2. 分片割裂语义：
- 缓解：窗口重叠 + Reduce 阶段跨片实体对齐。
3. 运行成本超预算：
- 缓解：视觉帧动态采样 + 工具调用配额 + 缓存。
4. 调试复杂度提升：
- 缓解：统一 trace_id，记录 chunk 级日志与 token/时延。

1. 架构图（旧版 vs 新版）
2. 状态字段变更表
3. 基线与升级后对比报表（质量/性能/成本）

## 8.3 回滚预案
1. 保留现有单图工作流编排作为 fallback。

3. 任一关键指标回退超过阈值时自动降级到 legacy。
