# Pencil 使用指南

这份文档是给当前项目做移动端原型时用的速查说明，重点不是覆盖 Pencil 的全部功能，而是帮助你在搭页面时快速判断每个面板项是干什么的、什么时候该改它。

## 1. 先理解右侧面板在做什么

右侧属性面板可以理解成一句话：

选中了谁，右侧就展示这个对象的身份、布局、尺寸、外观和导出方式。

如果你本来想改一个卡片，结果选中了卡片里的文字，那么右侧出现的设置会完全不一样。所以每次操作前，先确认当前选中的到底是外层 frame，还是里面的 text。

## 2. 顶部标题区

顶部会显示当前选中的对象名称，比如 `header2`。

这一行最重要的作用是帮你确认：

1. 你当前到底在修改哪一个对象。
2. 这个对象是不是你真正想改的那一层。
3. 如果样式怎么改都不对，往往是因为选错层了。

## 3. Create Component

这个按钮表示把当前对象创建成组件。

适合做组件的对象：

1. 会重复出现很多次的底部导航按钮。
2. 上传卡片、知识库卡片、列表项。
3. 你希望后面统一换样式的一组 UI。

不适合组件化的对象：

1. 一次性页面结构。
2. 只在当前页面出现、不会复用的装饰块。

经验上，先把页面搭通，再做组件化，效率更高。

## 4. Context

这个区域主要是当前对象的上下文信息。

通常可以帮助你判断：

1. 当前对象是否处在某个父容器中。
2. 当前对象是不是组件实例。
3. 当前对象和页面结构的关系。

它不是最高频操作区，但在复杂页面里很有用。

## 5. Alignment

这一排图标表示对象的对齐方式。

它解决的是：

1. 元素在父容器里靠左、居中还是靠右。
2. 元素在垂直方向靠上、居中还是靠下。

要注意，它是否生效，取决于父容器是不是布局容器。

一个简单判断：

1. 如果父容器是自动布局，Alignment 很重要。
2. 如果父容器是绝对定位，更多时候看的是 X/Y。

## 6. Position

Position 区域一般包括：

1. X：水平位置。
2. Y：垂直位置。
3. R：旋转角度。

它的本质是控制对象相对父容器的位置。

但有一个特别容易踩坑的点：

如果父容器开启了自动布局，子元素的位置通常主要由父容器控制，X/Y 不再是核心控制手段。

所以当你发现自己改了 X/Y 但页面没按预期走，先检查父容器是不是开了 Flex Layout。

## 7. Flex Layout

这是最核心的一块。它决定一个 frame 如何组织自己的子元素。

通常有三种状态：

1. 无布局：子元素自由摆放，主要靠 X/Y。
2. 垂直布局：子元素从上到下排列。
3. 水平布局：子元素从左到右排列。

这也是为什么两个按钮想横着排时，关键不是拖位置，而是把它们放进一个水平布局的父 frame。

### 7.1 九宫格 Alignment

这个九宫格控制子元素在容器里的对齐方式。

如果当前是垂直布局：

1. 主轴是上下。
2. 交叉轴是左右。

如果当前是水平布局：

1. 主轴是左右。
2. 交叉轴是上下。

你可以把它理解成“这一组元素在父容器里靠哪边站”。

### 7.2 Gap

Gap 是子元素之间的间距。

规则很简单：

1. 垂直布局时，Gap 是上下间距。
2. 水平布局时，Gap 是左右间距。

它是最值得优先使用的排版参数之一，因为比手动拖动更稳定。

### 7.3 Space Between / Space Around

这是主轴分布方式。

区别如下：

1. 普通 Gap：元素按顺序排列，只靠固定间距分开。
2. Space Between：首尾贴边，中间自动拉开。
3. Space Around：每个元素周围都有近似均匀的留白。

常见场景：

1. 标题栏左右分开，适合 Space Between。
2. 一组平均分布的操作按钮，可以考虑 Space Around。

## 8. Padding

Padding 是内边距，也就是内容距离容器边缘的距离。

这和 Gap 很容易混淆，区别是：

1. Gap 是子元素和子元素之间的距离。
2. Padding 是子元素和父容器边界之间的距离。

如果你做一张卡片：

1. 卡片内部标题和正文之间的距离，用 Gap。
2. 卡片内容不要贴边，用 Padding。

## 9. Dimensions

这一块控制宽高以及尺寸行为。

常见项目包括：

1. W：宽度。
2. H：高度。
3. Fill Width / Fill Height。
4. Hug Width / Hug Height。
5. Clip Content。

### 9.1 Fill Width / Fill Height

表示让元素沿对应方向撑满父容器。

适用场景：

1. 主按钮铺满整行。
2. 卡片铺满内容区域。
3. 需要跟随父容器尺寸变化的容器。

### 9.2 Hug Width / Hug Height

表示元素包裹内容。

适用场景：

1. 标签。
2. 小型按钮。
3. 胶囊 chip。
4. 只想刚好包住内部文字的容器。

如果你希望一个元素占满一行，但它始终缩在内容宽度上，通常就是因为它被设置成了 Hug Width。

### 9.3 Clip Content

表示裁切超出容器边界的内容。

适合：

1. 圆角卡片内裁图。
2. 抽屉、弹层、轮播这类需要边界裁切的区域。
3. 避免子元素溢出父容器。

## 10. Appearance

这一块通常控制整体显示属性，比如：

1. Opacity：透明度。
2. Corner Radius：圆角。

对于移动端 UI，圆角的使用频率很高。按钮、卡片、底部导航、输入框，通常都要在这里调圆角。

需要注意：

只给容器设置圆角，不代表内部内容一定会被裁切。
如果想让子元素也受圆角边界约束，往往还要打开 Clip Content。

## 11. Fill

Fill 是填充，也就是背景。

常见类型：

1. 纯色。
2. 渐变。
3. 图片填充。

如果你想让一个区域更像“主视觉”，最常见的方式就是给它一个更有层次的 Fill，比如渐变背景。

## 12. Stroke

Stroke 是描边。

常用场景：

1. 输入框边框。
2. 卡片的细描边。
3. 选中态边框。
4. 分段选择器边界。

在浅色界面中，很多层次感不是靠重阴影，而是靠很轻的 Stroke。

## 13. Effects

Effects 是效果，一般最常用的是阴影。

它常用于：

1. 让卡片浮起来。
2. 突出主操作按钮。
3. 让顶部 hero card 更有层次。

经验建议：

1. 阴影不要所有卡片都加。
2. 模糊半径可以大一点。
3. 透明度保持低一点。
4. 展示型主卡片可以略重，普通信息卡片尽量轻。

## 14. Export

Export 区域是导出当前对象。

常见用途：

1. 导出整页原型图。
2. 导出单个卡片给开发参考。
3. 导出图标或局部资源。

注意：

1. 选中哪个对象，就导出哪个对象。
2. 想导出整页，就先选中最外层页面 frame。

## 15. 最容易混淆的几组概念

### 15.1 Position 和 Flex Layout

1. Position 是手动摆坐标。
2. Flex Layout 是父容器自动排列孩子。

一旦父容器启用了自动布局，X/Y 通常不是主要控制方式。

### 15.2 Gap 和 Padding

1. Gap：孩子之间。
2. Padding：孩子和父容器边缘之间。

### 15.3 Fill Width 和 Hug Width

1. Fill Width：撑满父容器。
2. Hug Width：包裹内容。

### 15.4 Fill 和 Stroke

1. Fill：背景。
2. Stroke：边框。

### 15.5 圆角和 Clip Content

1. 圆角只影响容器形状。
2. Clip Content 决定内容是否也被边界裁切。

## 16. 做页面时推荐的操作顺序

如果你要搭一个卡片、一排按钮或者一个标题区，推荐按这个顺序操作：

1. 先确认选中的是外层 frame，而不是里面的 text。
2. 决定这个 frame 是无布局、垂直布局还是水平布局。
3. 设置 Gap。
4. 设置 Padding。
5. 决定宽高是 Fill 还是 Hug。
6. 再调 Fill、Stroke、圆角和阴影。

这个顺序会比一开始就调颜色、阴影更稳定，因为结构先对，视觉才好收。

## 17. 针对当前项目的实战建议

结合这个视频总结器项目，建议这样使用 Pencil：

1. 页面最外层 frame 通常固定为手机画板尺寸。
2. 页面主体内容区优先用垂直布局。
3. 顶部标题栏常用水平布局加 Space Between。
4. 一组横排按钮一定要先创建一个水平父容器，再把按钮放进去。
5. 主按钮一般使用 Fill Width。
6. 标签、小胶囊、模式切换项通常使用 Hug Width。
7. 卡片建议用轻描边加轻阴影，不要全部做成重投影。
8. 结果页、知识库页、会话页应统一使用同一套圆角和间距节奏。

## 18. 一个快速判断口诀

当你不知道该改哪里时，可以用下面这个判断顺序：

1. 位置不对：先看父容器是不是 Flex Layout。
2. 元素排不整齐：先看 Gap 和 Alignment。
3. 内容太贴边：看 Padding。
4. 元素太宽或太窄：看 Fill/Hug。
5. 区块没层次：看 Fill、Stroke、Effects。
6. 圆角没生效：看 Clip Content。

## 19. 最后一个经验

在 Pencil 里，很多问题不是“颜色不对”，而是“层级和布局没搭对”。

所以优先级应该始终是：

1. 先搭结构。
2. 再调尺寸。
3. 最后补视觉。

如果结构是对的，后续你不管做视频总结页、知识库页，还是侧边栏、时间旅行追问面板，都会顺很多。
